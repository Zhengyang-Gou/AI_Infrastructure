# 初识分布式训练

## 并行方法
多 GPU 配置对于加速训练以及将大型模型装入内存非常有效，否则单块 GPU 无法容纳这些模型。它依赖于将工作负载分摊到多个 GPU 上。并行化有几种类型，如：数据并行 (Data Parallelism)、张量并行 (Tensor Parallelism)、流水线并行 (Pipeline Parallelism) 和 模型并行 (Model Parallelism)。每种类型拆分负载的方式不同，有的拆分数据，有的拆分模型。

## 扩展策略
| 设置 (Setup) | 场景 (Scenario) | 策略 (Strategy) |
|---|---|---|
| 单节点 / 多 GPU | 模型能装进单块 GPU | DDP（分布式数据并行）或 ZeRO |
| 单节点 / 多 GPU | 模型装不下单块 GPU | 流水线并行 (PP)、ZeRO 或 张量并行 (TP) |
| 单节点 / 多 GPU | 模型中最大的那一层都装不下 | 张量并行 (TP) 或 ZeRO |
| 多节点 / 多 GPU | 节点间连接快（如 NVLink） | ZeRO 或 3D 并行（流水线并行、张量并行、数据并行的组合） |
| 多节点 / 多 GPU | 节点间连接慢 | ZeRO 或 3D 并行（流水线并行、张量并行、数据并行的组合） |

## Data Parallelism
数据并行将数据均匀地分布在多个 GPU 上，每个 GPU 都持有一个完整的模型副本，并同时处理各自那部分数据，最后来自每个 GPU 的结果会被同步并合并。

数据并行通过并行处理数据显著缩短了训练时间，并且可以根据可用 GPU 的数量进行扩展。然而，同步各 GPU 的结果会产生额外的通信开销。

数据并行有两种类型：DataParallel (DP) 和 DistributedDataParallel (DDP)。

### DataParallel
DataParallel 支持在带有多个 GPU 的单台机器上进行分布式训练。

- 默认 GPU（GPU 0）读取一个大批次的数据，并将其中一部分小批量发送给其他 GPU

- 最新的模型从 GPU 0 复制到其他 GPU

- 每个 GPU 执行前向传播，并将输出结果发送回 GPU 0 以计算损失

- 损失从 GPU 0 分发到其他 GPU，以进行反向传播

- 每个 GPU 的梯度被送回 GPU 0 并进行平均处理

### DistributedDataParallel
DistributedDataParallel 支持跨越多台机器（每台机器可有多个 GPU）进行分布式训练。

- 主进程将模型从默认 GPU 复制到每个 GPU

- 每个 GPU 直接处理一个小批量的数据（无需像 DP 那样由 GPU 0 分发）

- 在反向传播过程中，局部梯度会在所有 GPU 之间进行平均同步

推荐使用 DDP，因为它减少了 GPU 之间的通信开销，能更高效地利用每个 GPU，并且可以扩展到多台机器。

- 在 DP 中，GPU 0 像个苦力活领班，负责分发数据、汇总输出、计算 Loss、汇总梯度。这会导致 GPU 0 的显存占用远高于其他卡，且通信带宽容易遇到瓶颈

- DP 是单进程多线程。受限于 Python 的全局解释器锁 (GIL)，效率较低；DDP 是多进程。每个 GPU 都有一个独立的进程，互不干扰，完全绕过了 GIL

- DDP 使用了更先进的通信算法（如 Ring-AllReduce），梯度在计算的同时就在同步，不需要像 DP 那样全部堆到 GPU 0 处理

### ZeRO data parallelism
Zero Redundancy Optimizer 是一种显存效率更高的数据并行方式,它通过在各个数据并行进程之间对参数、梯度和优化器状态进行切分（分片存储），从而显著提高显存利用率并降低显存占用。

ZeRO 分为三个阶段：

- 只对优化器状态进行切分

- 对优化器状态和梯度进行切分

- 对优化器状态、梯度以及模型参数全部进行切分

显存消耗对比：

模型参数量 $\Psi = 7.5B$，使用 $N_d = 64$ 块 GPU 训练

- Baseline: 每块 GPU 都要存一份完整的参数、梯度和优化器状态   显存占用：120GB

- ZeRO-1: 仅切分优化器状态   显存占用：31.4GB

- ZeRO-2: 切分优化器状态 + 梯度   显存占用：16.6GB

- ZeRO-3: 全部切分   显存占用：1.9GB

通信开销：

- ZeRO-1 和 ZeRO-2：通信量与传统 DDP 一致，获得了巨大的显存收益，却没有增加网络负担

- ZeRO-3：通信量增加了 1.5x。因为参数也被拆散了，每次做前向或反向计算前，GPU 都需要从别的卡那里把需要的参数临时借过来，算完再丢掉

## Model parallelism
模型并行将模型分布在多个 GPU 上，拆分模型有多种方法，但最典型的方法是将模型的不同层分布到不同的 GPU 上。

- 在前向传播中： 第一块 GPU 处理一批数据，然后将其传递给下一块 GPU 上的下一组层

- 在反向传播中： 数据从最后一层传回第一层

模型并行是训练超大模型的有用策略。然而，GPU 利用率是不均衡的，因为在任一时刻通常只有一个 GPU 在工作。在 GPU 之间传递结果也会增加通信开销，并可能成为瓶颈。

### Pipeline parallelism
流水线并行在概念上与模型并行非常相似，但它更高效，因为它减少了 GPU 的空闲时间。

流水线并行不再等待每块 GPU 处理完一整批数据，而是创建了微批次。一旦一个微批次处理完成，它就会立即传递给下一块 GPU。这样，每块 GPU 就可以并发地处理部分数据，而无需等待其他 GPU 完全处理完一整个大批次。

流水线并行具有与模型并行相同的优点，但它优化了 GPU 利用率并减少了空闲时间。不过，流水线并行可能更复杂，因为模型可能需要重写为 nn.Sequential 模块序列。此外，它也无法完全消除空闲时间，因为最后一次前向传播仍必须等待反向传播完成。

### Tensor parallelism
张量并行将大型的张量计算分布到多个 GPU 上。张量被横向或纵向切分，每个切片由独立的 GPU 处理。每个 GPU 在其张量切片上执行计算，最后在结尾处进行同步以重建最终结果。

张量并行对于训练单卡显存无法容纳的巨大模型非常有效。它也更快、更高效，因为每个 GPU 可以并行处理其张量切片，并且可以与其他并行方法结合使用。不过，与其它方法一样，张量并行也会增加 GPU 之间的通信开销。

## Hybrid parallelism
通过组合多种并行方法，以实现更显著的显存节省，并更高效地训练拥有数千亿参数的模型。

### Data parallelism and pipeline parallelism
数据并行与流水线并行的组合，会将数据分布在不同的 GPU 组之间，并为了实现流水线并行，将每个小批量进一步划分为更小的微批次。

GPU 0 和 1 可以将微批次数据卸载给 GPU 2 和 3，从而减少空闲等待时间。这种方法通过降低 GPU 的空闲利用率，优化了并行数据处理。

### ZeRO data parallelism, pipeline parallelism, and model parallelism (3D parallelism)
数据并行、流水线并行和模型并行结合在一起，构成了 3D 并行，以优化显存和计算效率。

- 显存效率：通过将模型横跨多个 GPU 进行拆分，并将其划分为多个阶段来构建流水线。这允许 GPU 并行处理微批次数据，从而降低了模型、优化器状态和激活值的显存占用。

- 计算效率：通过 ZeRO 数据并行实现。在 ZeRO 模式下，每个 GPU 仅存储模型、优化器和激活值的一个切片。这使得数据并行节点之间拥有更高的通信带宽，因为通信可以独立进行，或者与其他流水线阶段并行运行。
